# Revenue Forecasting — SARIMA Time Series Analysis

This project applies **Seasonal ARIMA (SARIMA) time series modeling** to forecast daily revenue for a telecommunications company using 731 consecutive daily observations. The goal is to identify the underlying trend and seasonal structure of the revenue series and build a reliable forecasting model to support financial planning and customer retention strategy.

The analysis covers:

- **Exploratory Analysis:** Line plot of raw revenue series, stationarity evaluation using the Augmented Dickey-Fuller (ADF) test, and first-order differencing to achieve stationarity.
- **Time Series Diagnostics:** Trend analysis with linear regression, ACF/PACF plots, spectral density (periodogram), additive decomposition into trend/seasonal/residual components, and residual inspection.
- **Model Identification:** SARIMA order selection guided by ACF, PACF, and decomposition results — resulting in `SARIMA(0,1,1)(1,0,1,30)` with a linear trend component.
- **Forecasting:** 146-day forecast on the held-out test set (80/20 split) with confidence intervals, evaluated using RMSE.

> **Note on Dataset Availability**
> The dataset (`churn_clean.csv`) has been removed from this repository as it is proprietary and cannot be shared publicly. All visualizations and outputs generated during the analysis are retained in the `output/` folder for reference and presentation purposes.

---

## Key Findings

- The original revenue series is **non-stationary** (ADF p-value = 0.32) with a strong upward trend explaining **82.3% of variance** (R² = 0.8233)
- First-order differencing achieves stationarity (ADF p-value ≈ 0.00)
- Decomposition reveals a **strong trend** and a **weak but recurring ~30-day seasonal cycle**
- Selected model: **SARIMA(0,1,1)(1,0,1,30) with linear trend**
- Forecast RMSE: **2.43** (in revenue units) on the 146-day test set

---

## Model Summary

**Selected Model:** `SARIMA(0,1,1)(1,0,1,30)` with `trend='t'`

| Parameter | Value | Meaning |
|---|---|---|
| p = 0 | No AR term | PACF showed no strong cutoff |
| d = 1 | First-order differencing | ADF confirmed non-stationarity |
| q = 1 | MA(1) term | ACF spike at lag 1 with rapid decay |
| P = 1 | Seasonal AR | Recurring 30-day cycle in decomposition |
| D = 0 | No seasonal differencing | Regular differencing already stabilized series |
| Q = 1 | Seasonal MA | Cyclical error component |
| s = 30 | Season length | ~30-day cycle identified in decomposition |

**Diagnostics:**

| Test | Result | Conclusion |
|---|---|---|
| Ljung-Box (residual autocorrelation) | p = 0.14 | No significant autocorrelation remaining |
| Jarque-Bera (normality) | p = 0.49 | Residuals approximately normal |
| Heteroskedasticity | p = 0.42 | Variance stable over time |
| AIC / BIC / HQIC | 805.5 / 827.4 / 814.1 | Adequate model fit |

## How to Run

### Prerequisites

```bash
pip install -r requirements.txt
```

### Dataset Setup

Place the dataset inside the `data/` folder:

```
project/
├── data/
│   └── churn_clean.csv         ← put it here
├── main.py
├── requirements.txt
└── README.md
```

### Run

```bash
python main.py
```

The script handles all preprocessing, stationarity testing, diagnostics, model fitting, forecasting, and visualization automatically. All outputs are saved to the `output/` folder. Intermediate datasets are also saved to `data/`:

- `data/cleaned_churn_dataset.csv` — full cleaned series with datetime index
- `data/cleaned_churn_Train.csv` — training split (80%)
- `data/cleaned_churn_Test.csv` — test split (20%)
